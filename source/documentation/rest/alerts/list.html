GET alerts

List existing alerts for a given time range, optionally filtered by type (SMS, Email, Phone, or Push).

Resource URL

GET https://<subdomain>.pagerduty.com/api/v1/alerts

Parameters

This action is paginated. See the pagination documentation for details.
NameTypeRequiredDescription
sinceDateYesThe start of the date range over which you want to search. The time element is optional.
untilDateYesThe end of the date range over which you want to search. This should be in the same format as since. The size of the date range must be less than 3 months.
filter[type]StringNoReturns only the alerts of the said types. Can be one of SMS, Email, Phone, or Push.
time_zoneTime ZoneNoTime zone in which dates in the result will be rendered. Defaults to account time zone.

Examples

Alerts with Date Range

curl -H "Content-type: application/json" -H "Authorization: Token token=E7px6VVr3PVHZPJq51oa" -X GET -G \
    --data-urlencode "since=2013-03-06T15:28-05" \
    --data-urlencode "until=2013-03-06T15:30-05" \
    "https://acme.pagerduty.com/api/v1/alerts"

Note: Content-type of the request must be application/json
