PUT escalation_policies/:escalation_policy_id/escalation_rules/:id

Updates an escalation rule.

Resource URL

PUT https://<subdomain>.pagerduty.com/api/v1/escalation_policies/:escalation_policy_id/escalation_rules/:id

Parameters

NameTypeRequiredDescription
escalation_delay_in_minutesIntegerNoThe escalation timeout in minutes. If an incident is not acknowledged within this timeout then it will escalate onto the next escalation rule.
rule_objectObjectNoAn object containing the schedule or user id and type.

Rule Object Parameters

NameTypeRequiredDescription
typeStringNoA string of either schedule or user,
idStringNoThe id of the schedule or user assigned to this rule.

Escalation Rule Parameters

NameTypeRequiredDescription
escalation_delay_in_minutesIntegerNoThe escalation timeout in minutes. Must be at least 5 if the rule has multiple targets, and at least 1 if not. If an incident is not acknowledged within this timeout then it will escalate onto the next escalation rule.
rule_objectObjectNo
This field is deprecated. See targets instead
 A single target. Ignored if targets is provided.
targetsArrayNoThe array of the targets an incident should be assigned to upon reaching this rule. Parameters detailed below.

Target Parameters

NameTypeRequiredDescription
typeStringYesA string of either schedule or user,
idStringYesThe id of the schedule or user assigned to this rule.

Example

Update an escalation rule

curl -H "Content-type: application/json" -H "Authorization: Token token=E7px6VVr3PVHZPJq51oa" -X PUT \
    -d '{"escalation_delay_in_minutes":44,"targets":[{"type":"schedule","id":"PWEVPB6"}]}' \
    "https://acme.pagerduty.com/api/v1/escalation_policies/PBZ3BZV/escalation_rules/PNTPYK0"

Note: Content-type of the request must be application/json
