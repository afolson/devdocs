GET escalation_policies

List all the existing escalation policies.

Resource URL

GET https://<subdomain>.pagerduty.com/api/v1/escalation_policies

Parameters

This action is paginated. See the pagination documentation for details.
NameTypeRequiredDescription
queryStringNoFilters the result, showing only the escalation policies whose names match the query.

Response Fields

NameTypeDescription
escalation_policiesArrayThe collection of escalation policies objects returned by the query. Fields detailed below.

Escalation Policy Response Fields

NameTypeDescription
idStringThe ID of the escalation policy.
nameStringThe name of the escalation policy.
num_loopsIntegerThe number of times the escalation policy will repeat after reaching the end of its escalation.
escalation_rulesArrayA list of the policy's escalation rules in order of escalation. Fields detailed below.
servicesArrayA list of services using this escalation policy. See Services for the fields to expect.

Escalation Rule Response Fields

NameTypeDescription
idStringThe ID of the escalation rule.
escalation_delay_in_minutesIntegerThe amount of time before an incident escalates away from this rule.
rule_objectObject
This field is deprecated. See targets instead
 An object representing the first target.
targetsArrayAn array of targets which an incident will be assigned to upon reaching this rule. Fields detailed below.

Target Response Fields

NameTypeDescription
idStringThe id of the target.
typeStringA representation of the type of the target. Will be either schedule or user.
nameStringThe name of the target.
emailStringOnly for user targets. The user's email address.
time_zoneStringOnly for user targets. The user's personal time zone.
colorStringOnly for user targets. The color used to represent the user in schedules.

Example

Get all escalation policies

curl -H "Content-type: application/json" -H "Authorization: Token token=E7px6VVr3PVHZPJq51oa" -X GET -G \
    "https://acme.pagerduty.com/api/v1/escalation_policies"

Note: Content-type of the request must be application/json
