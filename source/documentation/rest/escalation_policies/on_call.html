GET escalation_policies/on_call

List all the existing escalation policies with currently on-call users.

If the start and end of an on-call object are null, then the user is always on-call for an escalation policy level.

Resource URL

GET https://<subdomain>.pagerduty.com/api/v1/escalation_policies/on_call

Parameters

This action is paginated. See the pagination documentation for details.
NameTypeRequiredDescription
queryStringNoFilters the result, showing only the escalation policies whose names match the query.

Example

Get an escalation rule

curl -H "Content-type: application/json" -H "Authorization: Token token=E7px6VVr3PVHZPJq51oa" -X GET -G \
    "https://acme.pagerduty.com/api/v1/escalation_policies/on_call"

Note: Content-type of the request must be application/json
