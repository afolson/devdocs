PUT users/:user_id/notification_rules/:id

Update an existing notification rule.

Resource URL

PUT https://<subdomain>.pagerduty.com/api/v1/users/:user_id/notification_rules/:id

Parameters

NameTypeRequiredDescription
start_delay_in_minutesIntegerNoNumber of minutes it will take for the notification rule to be activated (from the time the incident is assigned to the owning user) and an alert be fired.
contact_method_idStringNoThe id of the contact method

Example

curl -H "Content-type: application/json" -H "Authorization: Token token=E7px6VVr3PVHZPJq51oa" -X PUT \
    -d '{"notification_rule":{"contact_method_id":"PNGKILZ","start_delay_in_minutes":5}}' \
    "https://acme.pagerduty.com/api/v1/users/PPSFHH7/notification_rules/PHU1XBC"

Note: Content-type of the request must be application/json

Response Status: 200 OK 

{

      "email": "bob@acme.com",
      "address": "bob@acme.com",
      "send_short_email": false,
      "user_id": "PPSFHH7"
    }
  }
