POST services/:service_id/email_filters

Create a new Email Filter for the specified service.

Resource URL

POST https://<subdomain>.pagerduty.com/api/v1/services/:service_id/email_filters

Parameters

NameTypeRequiredDescription
subject_modeStringNoOne of always, match, no-match, which, respectively, means to not filter the email trigger by subject, filter it if the email subject matches the given regex, or filter if it doesn't match the given regex. Defaults to always.
subject_regexStringNoThe regex to be used when subject_mode is match or no-match. It is a required parameter on such cases.
body_modeStringNoOne of always, match, no-match, which, respectively, means to not filter the email trigger by body, filter it if the body email matches the given regex, or filter if it doesn't match the given regex. Defaults to always.
body_regexStringNoThe regex to be used when body_mode is match or no-match. It is a required parameter on such cases.
from_email_modeStringNoOne of always, match, no-match, which, respectively, means to not filter the email trigger by its from address, filter it if the email from address matches the given regex, or filter if it doesn't match the given regex. Defaults to always.
from_email_regexStringNoThe regex to be used when from_email_mode is match or no-match. It is a required parameter on such cases.

Example

curl -H "Content-type: application/json" -H "Authorization: Token token=E7px6VVr3PVHZPJq51oa" -X POST \
    -d '{"email_filter":{"body_mode":"no-match","body_regex":"sev 3"}}' \
    "https://acme.pagerduty.com/api/v1/services/P9312OW/email_filters"

Note: Content-type of the request must be application/json

Response Status: 201 Created 

{
  "email_filter": {
