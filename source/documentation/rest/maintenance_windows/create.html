POST maintenance_windows

Create a new maintenance window for the specified services. No new incidents will be created for a service that is currently in maintenance.

Resource URL

POST https://<subdomain>.pagerduty.com/api/v1/maintenance_windows

Parameters

NameTypeRequiredDescription
requester_idStringNoThe user id of the user creating the maintenance window. This is only needed if you are using token based authentication.
maintenance_windowObjectNoThe maintenance window object. See maintenance window parameters for details.

Maintenance Window Parameters

NameTypeRequiredDescription
start_timeDateYesThis maintenance window's start time. This is when the services will stop creating incidents. If this date is in the past, it will be updated to be the current time.
end_timeDateYesThis maintenance window's end time. This is when the services will start creating incidents again. This date must be in the future and after the start_time.
descriptionStringNoA description for this maintenance window.
service_idsArrayYesThe ids of the services that are affected by this maintenance window.

Example

curl -H "Content-type: application/json" -H "Authorization: Token token=E7px6VVr3PVHZPJq51oa" -X POST \
    -d '{    
      "maintenance_window": {
        "start_time": "2012-06-16T13:00:00-04:00Z",
        "end_time": "2012-06-16T14:00:00-04:00Z",
        "description": "Upgrading to new software",
        "service_ids": [
          "PF9KMXH"
        ]
      },
      "requester_id": "PP1565R"
    }' \
    "https://acme.pagerduty.com/api/v1/maintenance_windows"

Note: Content-type of the request must be application/json
